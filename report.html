Fisheries Integrated Modeling System

What is FIMS?

A flexible suite of software tools to support sustainable fishery management

  • Fisheries stock assessment1,2 at core.
  • Connects to ecosystem, climate, and economic models/data.
  • Flexible for innovative future modeling work.
  • Collaborative community effort.
  • Addresses numerous priorities.

FIMS landing page | FIMS GitHub repo

  • [1] Fishery stock assessment: The scientific process of collecting, analyzing, and reporting on the condition of a fish stock and estimating its sustainable yield.
  • [2] Stock assessment model: The mathematical and statistical techniques stock assessments use to analyze and understand the impact of fisheries and environmental factors on fish stocks.

FIMS path

  • FIMS modules are written in C++ and linked to R1 using Rcpp2
  • Template Model Builder3 serves as the engine for statistical inference of FIMS

1R: A programming language and free software environment for statistical computing and graphics.
2Rcpp: An R package that provides R functions as well as C++ classes which offer a seamless
integration of R and C++.
3Template Model Builder: An R package for fitting statistical latent variable models to data.

This Quarto file is both a guide and a template

Light and dark themes

Two color schemes following NMFS branding guidelines

Customizing the title slide

Background and swoop types can be changed

In the YAML header under title-slide-attributes:

  • data-background-image controls the swoop type and whether to include a background image

  • data-background-color controls the background color of the title slide if no background image is used

Customizing the title slide

Swoop type

data-background-image: "[Swoop Path]"

Customizing the title slide

Background image

data-background-image: "[Swoop Path], [Background Path]"
data-background-image: "styles/horiz-swoop-1.png, styles/background.jpg"

Customizing the title slide

Background image

data-background-image: "[Swoop Path], [Background Path]"
data-background-color: "{{< brand color dark-blue >}}"
Important

If using a background image that’s different from your theme (i.e., dark background in light theme, light background in dark theme), make sure to also change the data-background-color field to match the background so Quarto knows to use a contrasting font color

Customizing the title slide

Background color

For brand colors use {{< brand color [COLOR] >}}

data-background-color: "{{< brand color [COLOR] >}}"
